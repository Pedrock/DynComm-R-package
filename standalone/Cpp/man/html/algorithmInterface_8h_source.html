6  * Interface for main algorithms implemented in C++11.
7  *
8  * New algorithms must implement the functions defined in the class declared in
9  * this file. They should also extend the class "AlgorithmBase" which stores
10  * references to the graph, quality and program parameters objects.
11  *
12  * Copies of those objects must not be executed because the actual adding and
13  * removing from the graph is performed outside the control of the algorithm
14  * object the developer is implementing. For that same reason, the algorithm
15  * being implemented should not change the graph unless required.
16  *
17  * Due to optimization, and to relief the programmer of new algorithms from
18  * having to duplicate code, the functions used to add and remove edges are
19  * split into two: the addRemoveEdgePre and addRemoveEdgePost which are used,
20  * respectively, to process any data required before and after the edge is
21  * added/removed from the graph.
22  *
23  * The arguments passed to both functions are the same, not just identical.
24  *
25  * Actually adding/removing the edge from the graph is performed automatically
26  * between the call to addRemoveEdgePre and addRemoveEdgePost so the programmer
27  * does not have to be concerned with it.
28  *
29  * The call to function run also occurs automatically but, this time, after the
30  * last call to addRemoveEdgePost.
31  *
32  * See the implementation of the addRemoveEdges function of class "Algorithm"
33  * for details of when functions are called.
34  *
35  * Because all algorithms have an object of its class instantiated, even if it
36  * is not used and is never called, new algorithms should have a minimal
37  * memory footprint. This is implemented like so to improve performance by
38  * getting rid of object pointers and, consequently, of indirection and pointer
39  * resolution which would significantly slow down the program. Otherwise, every
40  * cycle in the program would require many function calls from object pointers.
41  *
